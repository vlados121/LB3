**Дисципліна:** Основи програмної інженерії  
**Проєкт:** AI-сервіс аналізу документів DocuCT

## 📌 Опис роботи
У рамках цієї лабораторної роботи було реалізовано програмний модуль `DocumentManager` для системи DocuCT та покрито його модульними автотестами.

Модуль містить бізнес-логіку з обробки виняткових ситуацій (валідація формату, розміру файлів, розрахунок часу обробки ШІ тощо).

## 🛠 Технології та інструменти
* **Мова програмування:** Python 3
* **Фреймворк для тестування:** `pytest`
* **Інструмент перевірки покриття:** `coverage.py` (pytest-cov)

## 🧪 Проєктування тестів
Тести написані за патерном **AAA (Arrange - Act - Assert)**.  
Для оптимізації тестового набору застосовано формальні техніки тест-дизайну:
1. **EP (Equivalence Partitioning)** — еквівалентне розбиття (позитивні та негативні сценарії).
2. **BVA (Boundary Value Analysis)** — аналіз граничних значень (тестування лімітів розміру файлу та кількості сторінок).

Всього реалізовано **11 тест-кейсів**.

## 📊 Результати покриття (Code Coverage)
Тести забезпечують **100% покриття рядків коду (Line Coverage)**.

### Як запустити тести локально:
Щоб відтворити результати тестування, виконайте наступні команди в терміналі:

1. Встановіть залежності:
```bash
pip install pytest pytest-cov
```
2. Запустіть тести з генерацією HTML-звіту покриття:
```
pytest --cov=. --cov-report=html
```
Після виконання команди звіт буде доступний у папці htmlcov/index.html.
